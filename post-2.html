Why clamp() for type?

clamp(min, preferred, max) gives you fluid, viewport-aware sizes that never get too small or too big. It replaces brittle vw formulas and most typography media queries. Your homepage already uses this approach via tokens like --step-0, --step-1, etc.

Rule of thumb: Set min for small phones (readability), max for wide desktops (layout restraint), and make the middle term a small vw slope that feels natural during resize.

Drop-in Scales

Copy any of these into :root (you can mix & match). They’re tuned for comfortable reading widths.

Compact Scale (content-heavy UIs)

:root{
  --step--1: clamp(0.85rem, 0.80rem + 0.20vw, 0.95rem);
  --step-0:  clamp(1.00rem, 0.92rem + 0.40vw, 1.125rem);
  --step-1:  clamp(1.15rem, 0.98rem + 0.70vw, 1.35rem);
  --step-2:  clamp(1.35rem, 1.05rem + 1.10vw, 1.65rem);
  --step-3:  clamp(1.70rem, 1.20rem + 1.80vw, 2.20rem);
}

Reading Scale (articles/blogs)

:root{
  --step--1: clamp(0.95rem, 0.88rem + 0.25vw, 1.05rem);
  --step-0:  clamp(1.10rem, 0.98rem + 0.45vw, 1.25rem);
  --step-1:  clamp(1.35rem, 1.08rem + 0.85vw, 1.60rem);
  --step-2:  clamp(1.65rem, 1.20rem + 1.30vw, 2.00rem);
  --step-3:  clamp(2.10rem, 1.45rem + 2.10vw, 2.80rem);
}

Promo Scale (hero/marketing)

:root{
  --step--1: clamp(0.95rem, 0.85rem + 0.30vw, 1.10rem);
  --step-0:  clamp(1.10rem, 0.95rem + 0.50vw, 1.30rem);
  --step-1:  clamp(1.50rem, 1.10rem + 1.20vw, 1.95rem);
  --step-2:  clamp(2.00rem, 1.30rem + 2.20vw, 2.80rem);
  --step-3:  clamp(2.75rem, 1.60rem + 3.20vw, 3.60rem);
}

Plug into elements

body{ font-size: var(--step-0); line-height: 1.6; }
h1{ font-size: var(--step-3); line-height: 1.15; }
h2{ font-size: var(--step-2); line-height: 1.2; }
h3{ font-size: var(--step-1); line-height: 1.25; }
small, .caption{ font-size: var(--step--1); }

How to pick min / fluid / max

  1. Set min (phone-first): try 1rem body, 1.6–1.8 line-height.
  2. Set max (desktop cap): avoid billboard text; cap headings around 2.75–3.5rem.
  3. Choose slope: start with 0.4–1.4vw depending on how much growth you want.
Shortcut: If you like your current tokens, keep them and only tweak the middle term (the vw). Small nudges (±0.2vw) go a long way.

Make your own with a mixin (plain CSS)

/* usage: var(--fs-1) becomes clamp(min, fluid, max) */
:root{
  --fs-min: 320;   /* px viewport where min applies */
  --fs-max: 1280;  /* px viewport where max applies */
}
/* generate a token */
:root{
  --fs-1: clamp(1.1rem,
               calc(1.1rem + (1.6rem - 1.1rem) * ((100vw - var(--fs-min)*1px) / (var(--fs-max) - var(--fs-min)))),
               1.6rem);
}

Accessibility & safeguards

  • Readable widths: keep text blocks around 60–75ch (your blog already uses ~75ch).
  • Respect user zoom: use rem units so system font scaling works.
  • Motion safe: you already honor prefers-reduced-motion—great for UX.
  • Contrast: ensure color tokens meet WCAG AA (≥ 4.5:1 for body).

Usage patterns

Landing hero

.hero h2{ font-size: var(--step-3); }
.hero p{ font-size: var(--step-1); }

Cards / grids

.card h3{ font-size: var(--step-1); }
.card p { font-size: var(--step-0); }

Meta / captions

.meta, small{ font-size: var(--step--1); }
